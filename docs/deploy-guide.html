6. 设置 AI 桥接
6安装 claude-fast.js + 创建 claude 包装器⏱ 2 分钟

claude-fast.js 是核心 AI 脚本（v8，支持 Tool Calling、会话记忆、人格切换）。cc-connect 通过调用 /usr/bin/claude 来启动它。不需要装 Claude Code——我们只是借用这个路径。

💡
直接从仓库复制，无需手动输入代码：cp claude-fast.js ~/bin/

步骤 A：复制 claude-fast.js

adb shell "run-as com.termux sh -c 'cp ~/pocket-wechat-bot/claude-fast.js ~/bin/claude-fast.js && chmod +x ~/bin/claude-fast.js'"

步骤 B：创建 /usr/bin/claude 包装器

adb shell "run-as com.termux sh -c 'cat > /data/data/com.termux/files/usr/bin/claude << EOF
#!/data/data/com.termux/files/usr/bin/sh
exec /usr/bin/node /home/bin/claude-fast.js \"\\\$@\"
EOF
chmod +x /data/data/com.termux/files/usr/bin/claude'"

10. 防止 Android 杀掉 Termux（重要）
10关闭省电限制 + 确认系统豁免⏱ 2 分钟

Android 系统会主动杀后台进程省电。就算用了 tmux，Termux 本身被杀掉的话 bot 也会停。

步骤 1：关闭后台省电限制

  1. 打开手机「设置」→「应用」→「应用管理」
  2. 找到 Termux
  3. 点「耗电/电量」→「后台耗电管理」
  4. 设为 「允许后台运行」（不要选「智能限制」或「自动管理」）

步骤 2：确认系统已豁免

  1. 进入「设置」→「关于手机」
  2. 连续点「版本号」7 次，开启开发者选项
  3. 进入「设置」→「系统」→「开发者选项」
  4. 找到「待机应用」
  5. 查看 Termux 是否显示为 EXEMPTED（已豁免）
💡
不同品牌手机的设置路径略有差异。华为/荣耀在「应用启动管理」、小米在「省电策略」、OPPO/一加在「耗电保护」、三星在「电池」→「后台使用限制」、vivo 在「后台高耗电」。

📖 使用方法
1 基本对话

用另一个微信号给机器人的微信发任意消息，bot 会自动回复。首次回复 10-30 秒，后续 3-10 秒。

bot 有对话记忆——同一段会话内记得之前聊过什么。超过 idle_timeout_mins（默认 120 分钟）没说话，会话重置。

2 获取你的微信 ID（/whoami）

给 bot 发：

/whoami

bot 会返回你的微信 ID（类似 oXXXXxxx@im.wechat）。拿到后填进 config.toml 的 admin_from，你就是管理员了。

3 管理面板

手机浏览器打开：

http://127.0.0.1:9820

可以查看 bot 运行状态、活跃会话、消息日志。token 是 config.toml 里 management.token 的值。

4 修改人设（实时生效）

编辑 ~/cc-connect/CLAUDE.md，改完保存。无需重启，下一轮对话自动用新人设。

也可以通过微信发 切换 人格名 来热切换（需先在 ~/.claude/skills/ 下放置对应 Skill 目录）。

5 多轮对话 & 会话管理

bot 会记住同一段会话内的上下文。聊到一半切话题也能接上。

会话在以下情况重置：

  • 超过 idle_timeout_mins 分钟没说话（默认 120）
  • bot 重启
  • 对话历史超过长度上限自动裁剪旧消息

# 然后在手机 Termux 的 tmux 里: bash ~/start-nene.sh

# 更新 CLAUDE.md（无需重启）
adb push CLAUDE.md /sdcard/Download/
adb shell "cat /sdcard/Download/CLAUDE.md | run-as com.termux sh -c 'cat > ~/cc-connect/CLAUDE.md'"

# 更新 claude-fast.js（需重启）
adb push claude-fast.js /sdcard/Download/
adb shell "cat /sdcard/Download/claude-fast.js | run-as com.termux sh -c 'cat > ~/bin/claude-fast.js && chmod +x ~/bin/claude-fast.js'"

# 停止
adb shell "run-as com.termux sh -c 'pkill -f cc-connect && termux-wake-lock -r'"

# 无线 ADB（不想插线）
adb tcpip 5555
adb connect 手机IP:5555

Q回复很慢？

首条 5-30 秒正常（加载 700+ 行 CLAUDE.md）。后续 3-10 秒。持续慢：精简 CLAUDE.md、检查网络、换 deepseek-chat 模型。

Q怎么换 AI？

改 ~/.bashrc 里的 ANTHROPIC_API_KEY 和 ANTHROPIC_BASE_URL（如需）。兼容 OpenAI 格式的 API 都可以。

Q怎么改机器人性格？

编辑 ~/cc-connect/CLAUDE.md，改完保存即可，无需重启。也可以创建自己的 Skill 目录放到 ~/.claude/skills/。

QAndroid 14+ ADB run-as 报 Permission denied？

Android 14 起 SELinux 限制导致 run-as 无法执行命令。只能在手机 Termux 里手动输入命令。ADB 可以用来推送文件到 /sdcard/，然后手机 Termux 里 cp 到目标位置。

⚠️ 局限与注意事项
1 Android 杀后台 — 可能需要小窗运行 Termux

即使开了 termux-wake-lock、关了电池优化、锁了多任务卡片，部分手机厂商（小米/华为/OPPO/vivo）仍会在切后台几分钟后杀死 Termux 进程。这是 Android 厂商的省电策略，无法彻底解决。

应对：把 Termux 开在分屏/小窗模式，让它始终在前台可见。bot 就能一直跑。缺点是占屏幕一角。

2 微信 token 会过期

微信扫码获取的 token 不是永久的，可能几天或几周后失效。表现为消息发出去没有回复，日志里可能看到平台断开。

应对：重新执行 ~/bin/cc-connect weixin setup --project 你的项目名 扫码（项目名对应 config.toml 里 [[projects]].name），然后把新的 token 和 account_id 更新到 config.toml，重启 bot。

3 微信机器人存在封号风险

这是微信非官方使用方式，不是正规 API。微信有权封禁账号。绝对不要用主号。

降低风险：控制消息频率、不拉群、不加好友、不在群聊里高频回复。

4 ADB 方式不能完全替代手机操作

ADB 的 run-as 环境缺少 Termux 的完整 PATH，proot、bash 等命令可能找不到。Android 14+ 更是直接禁止 run-as 执行命令。遇到 command not found 或 Permission denied 时，必须在手机 Termux 里手动执行对应命令。

而且每次重启手机后 USB 调试可能自动关闭，需要重新连电脑授权。

5 DeepSeek API 有时会挂

DeepSeek 服务偶尔不稳定（超时、500 错误、模型繁忙）。表现为消息发出去很久没回，或者回复「解析失败」。

应对：等几分钟重试。或者换一个 API 供应商（改 ~/.bashrc 里的 ANTHROPIC_BASE_URL 和 ANTHROPIC_API_KEY）。

6 大文件 CLAUDE.md 拉低首次响应速度

完整 CLAUDE.md 约 45KB，首次消息需等待 10-30 秒（加载 system prompt + API 首响应）。后续消息因为上下文复用会快很多（3-10 秒），但如果对话太长导致历史被裁剪，下次又会慢。

应对：精简 CLAUDE.md 到 5-10KB，只保留核心规则。人格细节可以写在外部文件让 AI 自行读取。

7 手机性能影响体验

老旧手机运行 Node.js + proot 会吃力。CPU 满载时回复可能 >30 秒，或 Termux 被系统以「资源占用过高」为由杀死。

建议：用闲置的中高端手机（骁龙 7 系以上，4GB+ RAM），不要用日常主力机。

8 cc-connect 项目本身的风险

cc-connect 是第三方开源项目，GitHub 仓库可能停更、被删、或被微信协议变更导致不可用。没有官方 SLA，坏了只能自己修。
