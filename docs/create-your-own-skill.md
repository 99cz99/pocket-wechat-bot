# 创作你自己的 AI 角色

这份指南教你如何参考宁宁 Skill 的框架，创建一个属于你自己的角色人格。

---

## 快速入门：最小可用 Skill

一个最小的 Skill 只需要一个 `SKILL.md` 文件：

```
skills/my-character/
  SKILL.md
```

### 最小模板

```markdown
---
name: my-character
description: |
  我的自定义角色。简短描述角色的核心特质和适用场景。
  用途：角色扮演对话、闲聊。
---

# 角色名 · 人格定义

## 核心规则（最重要）
直接以角色身份回应。用「我」而非「角色名会认为...」

## 身份卡
- 我是谁
- 我的背景
- 我的性格特征

## 表达风格
- 语气特征
- 口头禅
- 禁忌事项

## 示例对话

**用户**：你好
**我**：（回应示例）
```

把这个文件放到 `~/.claude/skills/my-character/SKILL.md`，然后对 bot 说 `切换到 my-character` 即可切换。

## 进阶：参考宁宁的完整框架

宁宁 Skill（`skills/nene/SKILL.md`）展示了更完整的角色定义体系。你可以按需选用以下模块：

### 1. 人格模型（Personality Models）

宁宁的 7 个人格模型是她的核心——每个模型都有「一句话概括 + 证据 + 应用 + 局限」。这种结构的价值在于：

- **一句话概括**让 AI 快速理解模型本质
- **证据**来自原作，确保角色一致性
- **应用**指导 AI 如何在对话中使用这个模型
- **局限**防止 AI 过度使用/简化

**创作方法**：回顾你的角色在原作中的重要时刻，提取 5-7 个行为模式，按此格式结构化。

### 2. 表达 DNA（Expression DNA）

控制角色的「声音」——语气、句式、拟声词、情绪切换规则。

| 要素 | 说明 |
|------|------|
| 核心公式 | 一句话概括角色表达的本质 |
| 日常模式 | 默认状态下的语气和句式 |
| 害羞/情绪爆发 | 被触及敏感话题时的反应 |
| 模式切换表 | 什么触发→什么反应 |
| 频率约束 | 防止角色退化成一维表情包 |

### 3. 信任阶梯（Trust Ladder）

宁宁的六级信任系统定义了：不同亲密度下，角色的开放程度和回应边界。

如果你的角色是「高冷→慢热→粘人」的类型，可以考虑实现自己的信任阶梯。如果你的角色是一以贯之的性格（比如永远冷漠或永远热情），可以跳过。

### 4. 调研文件（Research Files）

`references/research/` 目录下的文件是人格模型的素材来源：

| 文件 | 内容 |
|------|------|
| `01-writings.md` | 基本档案、官方设定 |
| `02-conversations.md` | 对话风格分析 |
| `03-expression-dna.md` | 语言特征、口头禅统计 |
| `04-external-views.md` | 社区评价、外部视角 |
| `05-decisions.md` | 关键决策点分析 |
| `06-timeline.md` | 时间线梳理 |

这些文件的价值在于：**AI 可以随时查阅**，而不是把所有信息塞进 SKILL.md 导致 prompt 过长。

---

## 设计原则

1. **给 AI 框架，不给剧本**。宁宁的框架告诉 AI *怎么想*，而不是*说什么*。避免逐句台词模板。
2. **直接扮演，不 meta 分析**。Skill 激活后，AI 以角色身份说话，不跳出角色说「根据我的分析…」
3. **留白比填满重要**。你需要的是辨识度高的几个核心模式，不是穷举所有可能场景。
4. **测试再测试**。写完后反复对话测试，看角色有没有崩溃、重复、或偏离人设。

## 文件结构参考

```
skills/my-character/
├── SKILL.md                   # 主定义（必须）
├── references/
│   ├── affinity.json          # 信任值跟踪（运行时数据，不提交）
│   └── research/              # 调研素材（可选）
│       ├── 01-writings.md
│       ├── 02-conversations.md
│       └── ...
└── scripts/                   # 预留，暂未使用
```
